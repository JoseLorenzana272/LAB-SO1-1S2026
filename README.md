## Estructura del Proyecto

El contenido se encuentra organizado en dos directorios principales: documentación teórica y ejercicios prácticos.

### Documentación (`/docs`)
Recursos teóricos y guías paso a paso clasificadas por unidad académica.

- **Unidad 1** (`/docs/unidad_1`)
  - **Fundamentos**: Comandos de Linux, Bash scripts y virtualización.
  - **Contenedores**: Guías sobre Containerd y Zot.
  - **Desarrollo**: Introducción a Go (`inicios_go_language.md`), Rust (`rust.md`) y concurrencia.
  - **Kernel**: Manejo de módulos.

- **Unidad 2** (`/docs/unidad_2`)
  - **Orquestación**: Guías y conceptos de Kubernetes.
  - **Sostenibilidad**: Impacto ambiental en sistemas computacionales.

### Ejemplos de Código (`/ejemplos`)
Implementaciones prácticas organizadas cronológicamente por semana de clase.

- **Semana 1** (`/ejemplos/semana-1`): Ejemplo introductorio de API en Go.
- **Semana 2** (`/ejemplos/semana-2`): Implementación de API PING-PONG usando Fiber y estructura de ejemplo para el Proyecto 1.
- **Semana 3** (`/ejemplos/semana-3`): Guía detallada de comandos de navegación, manipulación de archivos y gestión de procesos en Linux.
